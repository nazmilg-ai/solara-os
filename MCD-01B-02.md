## Reference Implementation Note

This document follows the reference structure established by MCD-01B-01. Per the design brief for this specification (claude-code-brief-MCD-01B-02.md), Vertical Blinds exhibits materially more dynamic configuration behaviour and supplier variability than Roller. Where that richer behaviour requires more structure than MCD-01B-01 needed (e.g. §4 Louvre Width, §6 Dynamic Configuration Workflow, §21 Product Configuration Summary), this document adds sections rather than compressing genuine complexity to fit the Roller template. This is a documentation decision made within MCD-01B-01's established writing standard, not an architectural one — the Shared Service Profile, status definitions, and layering rules remain fully governed by MCD-01A v1.5, unchanged.

## Documentation Conventions: Advisor Selection / Derived Default / Supplier Capability

Per the Architectural Constraints in the design brief, the following three terms are used consistently throughout this document as **documentation conventions only** — they are not new architectural layers and do not modify MCD-01A §3:

- **Advisor Selection** — a value the advisor actively chooses. Architecturally, this is the Configuration layer (or, where the choice changes construction, the Construction/System layer).
- **Derived Default** — a value the system determines automatically from an Advisor Selection, without a separate question. Architecturally, this is not a new layer; it is the Accessory Engine and/or Supplier Capability Engine resolving a compatible component or value from an existing Configuration selection (the same mechanism MCD-01B-01 §11 uses for Bracket Type → physical bracket).
- **Supplier Capability** — the actual data (colours, compatibility, pricing, availability) that makes a Derived Default resolvable. Architecturally, this is the Supplier Capability Engine (§5.9) and the Supplier Capability Library, per MCD-01A §9.

Every advisor-facing question, automatic behaviour, and supplier-owned fact in this document is labelled with one of these three terms so the distinction is never ambiguous to a reader or implementer.

## 1. Purpose

This document defines the Vertical Blind implementation of the Sales MVP Product Engine.

It implements the architecture defined in MCD-01A. It shall not redefine any Shared Platform Service. Where shared functionality exists, this document references MCD-01A and defines only the Vertical-specific implementation.

**Included — Construction Variants**
- Slimline
- Nova
- Cruze
- Vogue
- Curved Vertical (specialist)
- Sloping Vertical (specialist)

**Excluded** — the following are not separate Vertical products:

**Future**

## 3. Construction Variants

**Terminology note:** consistent with the precedent set in MCD-01B-01 §3, "Construction Variant" is this specification's name for the concept MCD-01A §8 calls "Product Variants" — genuine construction differences only, never Material/Fabric attributes. This is not re-derived here; it is inherited directly from the Roller reference implementation.

**Headrail System constructions** — differ by headrail mechanism only, not by fundamental product type:
- **Slimline** — compact headrail system.
- **Nova** — standard headrail system.
- **Cruze** — standard headrail system.
- **Vogue** — premium headrail system.

**Specialist constructions** — genuinely different manufacturing routes, requiring supplier-specific geometry data not detailed in this document (see §8, §20):
- **Curved Vertical** — curved track geometry.
- **Sloping Vertical** — sloping/angled head geometry.

Construction differences only. Material selection, Control Type, and Headrail Colour shall not create additional Construction Variants.

## 4. Louvre Width

Sales MVP exposes **89mm louvre width only**. 127mm is retained as **Legacy/Future** capability and shall not appear in the normal advisor workflow — it is not a Survey question (§6), not an active Validation value (§8), and not offered in the Product Configuration Summary (§21).

**Business reason:** very low market demand; declining supplier fabric availability.

**Shared Service Profile treatment:** this deliberate exclusion is reflected in §5's Validation Engine and Supplier Capability Engine rows (both remain **Configured**, with an explicit note), rather than being implied by omission. The reasoning for classifying it this way — rather than marking either row **Restricted** — is set out in full in the Architectural Review Question response, logged in §23 Decision Register (Question A). Future Product Specifications facing the same documentation question should reference that entry rather than re-deriving it, per the brief's Decision Register Requirement.

### Dynamic Configuration Workflow

This subsection documents the conditional branching the Survey Engine's existing "question sequencing" and "conditional branches" responsibilities (MCD-01A §5.1) are configured to run for Vertical. It is Configured behaviour, not Extended — the Survey Engine already supports conditional sequencing; this is Vertical-specific parameterisation of it.

**Control Type** (Advisor Selection) — exactly three values. No additional terminology (e.g. "Wand & Cord," "Split Wand," "Mono Control") shall be introduced; these are Supplier Capability implementation details, not Product Engine terms.

- **Wand selected** → display a single Draw Configuration selection: Wand Left / Wand Right / Split Draw. **Confirmed product rule:** for Wand operation, control position and stack direction are physically coupled by the headrail design — Wand Left means the wand is positioned left and the blind stacks left; Wand Right means the wand is positioned right and the blind stacks right. There is no separate Control Side question for Wand operation. Split Draw is a centre split; whether the system uses a single centre wand or two wands is Supplier Capability (§20); the advisor does not choose this.
- **Cord & Chain selected** → display Control Side (Left / Right), then a separate Draw Configuration: Left Stack / Right Stack / Split Draw. These are two separate Advisor Selections, not one.
- **Motorised (Tilt Only) selected** → display Motor Left / Motor Right (tilt motor position only). Applies to louvre tilt only; there is no motorised draw. **Confirmed rule (Vogue Construction Variant):** draw is performed via the operating wand automatically when Control Type = Motorised (Tilt Only) and Construction Variant = Vogue — a Supplier Capability manufacturing fact, not an advisor choice or Derived Default in the selection sense, so no separate draw-mechanism question is presented for this combination. For other Construction Variants, the compatible draw mechanism for Motorised (Tilt Only), where offered, is determined by Supplier Capability (§20). Full evidence and classification are recorded in §23 Decision Register.

## 7. Construction Variant Compatibility Note

Curved Vertical and Sloping Vertical introduce additional Supplier-Capability-driven survey requirements beyond the standard workflow above (specialist geometry inputs). These are not enumerated here, consistent with keeping supplier-specific implementation out of the Product Specification (MCD-01A §9); see §20.

## 8. Validation Rules

The shared Validation Engine performs validation. Vertical-specific validation includes:
- Minimum width
- Maximum width
- Minimum drop
- Maximum drop
- Construction Variant compatibility
- Control Type compatibility
- Draw Configuration compatibility
- Headrail Colour availability (per Construction Variant and supplier)
- Bracket compatibility
- Motor compatibility (tilt motor only)
- Fabric compatibility
- Supplier manufacturing limitations
- Motorised (Tilt Only) width limits (Vogue-specific — see confirmed values below)
- Curved/Sloping specialist validation (geometry-specific; supplier-defined)

**Deliberate business restriction:** louvre width validation accepts 89mm only in Sales MVP; 127mm is a Legacy/Future value, not validated as an active option. See §4 and §23.

**Confirmed data — Motorised (Tilt Only) on Vogue:** maximum width 3m; minimum width 200mm where an open cassette is used. These limits apply specifically to Motorised operation on the Vogue Construction Variant — they are not general Vogue width limits and do not apply to Manual (Wand or Cord & Chain) operation on Vogue. See §23 Decision Register for source evidence.

Supplier-specific values are not otherwise defined here.

**Not yet specified — needs supplier/business data before technical design:** minimum/maximum drop values, minimum/maximum width values for combinations other than Motorised (Tilt Only) on Vogue (confirmed above), and the exact Curved/Sloping geometry validation parameters. These require supplier data and are flagged here rather than invented, consistent with the treatment MCD-01B-01 §7 gives its equivalent open items.

## 11. Operation & Control Types

Operation behaviour is defined by the shared Operation Engine.

**Vertical supports:**
- Manual (Wand or Cord & Chain — see §6)
- Manual or Motorised (Advisor Selection) — motorisation applies to louvre tilt only

**Vertical does not support:**
- Manual or Motorised (Factory Selection)
- Motorised Only

Operation modes shall not be redefined.

**Vertical-specific scope note:** unlike a product where "motorised" applies to the whole operating function, Vertical's motorisation is deliberately scoped to tilt only — draw/traverse remains manual in every configuration, including when tilt is motorised. This is a product-specific implementation detail (which physical function the shared "motorised" concept applies to), not an architectural ambiguity — the Operation Engine's four modes are unaffected; only the physical scope of "motorised" is Vertical-specific, which is exactly what the Operation Engine's extension point ("Product Specifications define only supported operation methods") permits.

## 12. Motor Implementation

Motor compatibility is determined by the shared Motor Engine. Compatible tilt-motor families are determined by the Supplier Capability Library — refer to the applicable Supplier Capability document for supported motor systems. No motor suppliers are named in this specification.

Motor selection depends upon: Supplier, blind dimensions, tilt mechanism, customer requirements, power availability.

Motor compatibility shall not be hard-coded.

## 13. Headrail Colour and Derived Defaults

**Headrail Colour** is an Advisor Selection (Configuration layer). Available colours are determined by Supplier Capability.

Examples verified so far: Nova (White, Grey, Black); Vogue (White, Grey, Anthracite, Brown, Black, Brushed Aluminium, Champagne Gold). Remaining colour ranges (e.g. Cruze, Slimline) are Supplier Capability and are not named here beyond these verified examples.

**Derived Defaults (from Headrail Colour):** Wand colour, Cord colour, Bracket colour, end caps, and other matching visible hardware are all Derived Defaults — determined automatically the moment Headrail Colour is selected. The advisor is never prompted separately for these. Architecturally, this is the Accessory Engine and Supplier Capability Engine resolving the matching component from the Configuration value already selected — the same mechanism as MCD-01B-01 §11's Bracket Type reconciliation, applied here to colour-matching instead of bracket eligibility. It is not a new architectural layer (see Documentation Conventions, above).

## 14. Brackets

The advisor selects **Bracket Type only** (Advisor Selection — e.g. Face Fix, Top Fix, Extension Bracket). Bracket colour is a **Derived Default** from Headrail Colour (§13); the advisor never selects bracket colour. Supplier Capability determines the correct physical bracket.

This is the same Configuration-to-Accessory relationship established as precedent in MCD-01B-01 §11: Bracket Type is Configuration (a mandatory selection that doesn't change the underlying construction); the specific bracket products are Accessory Engine items whose eligibility Bracket Type and Construction Variant jointly determine. Not re-derived here — referenced directly per the Decision Register Requirement's precedent principle.

## 15. Chain, Bottom Weights, and Bottom Chain

**Terminology (confirmed product decision):** "Bottom Weights" and "Bottom Chain" are the standardised Product Engine terms for these two components. "Bottom Stabilising Chain" is not used as internal terminology in this document and shall not be reintroduced, except when directly quoting supplier literature, should that ever occur. This was previously logged as an open interpretation (§23 Decision Register); it is now a confirmed decision, not an open item.

**Chain** (operating chain, Cord & Chain Control Type only) — Advisor Selection: White Plastic (Standard) or Steel Chain (Optional Upgrade). Pricing supplied by Supplier Capability. Not motorised (Motorised Control Type has no operating chain).

**Bottom Weights** — the plastic weights inserted into every vane. These always exist, regardless of Control Type. Advisor Selection: White (Standard) or Black (Optional Upgrade). Pricing supplied by Supplier Capability.

**Bottom Chain** (the chain joining the bottom weights together) — Advisor Selection: White, Black, or No Chain.
- If **No Chain** is selected, the manufacturer does not simply remove the chain — a different bottom weight system designed for chainless operation is used instead. The advisor never chooses the weight type in this case; the substitution is a **Derived Default**, resolved automatically by the Supplier Capability Engine.

## 16. Accessories

The shared Accessory Engine manages compatibility. Vertical accessory groups include:
- Universal Brackets
- Top Fix Brackets
- Face Fix Brackets
- Extension Brackets
- Wands
- Chain (operating, Cord & Chain only)
- Bottom Weights
- Bottom Chain
- Child Safety Devices
- Remote Controls (Motorised tilt only)
- Wall Switches (Motorised tilt only)
- Smart Hubs (Motorised tilt only)

**Bracket Type reconciliation:** see §14 — Bracket Type is the Configuration-layer selection; the specific bracket products above are Accessory Engine items whose eligibility Bracket Type and Construction Variant jointly determine.

**Bottom Chain mandatory-substitution rule:** see §15 — when Bottom Chain = No Chain, the Accessory Engine substitutes the compatible chainless bottom weight system automatically; this is not an advisor-facing accessory choice.

**Control Type reconciliation:** Control Type (§6) is the Configuration-layer selection. It determines which control-related Accessory Engine items become eligible: Wand operation enables the compatible wand components; Cord & Chain operation enables compatible cord and chain components; Motorised Tilt enables the compatible motor-control accessories. The advisor selects the Control Type; the Accessory Engine and Supplier Capability Engine determine the eligible physical components.

**Chain Type reconciliation:** Chain Type (§6) is a Configuration-layer selection shown only when Cord & Chain operation is selected. It determines which compatible physical chain item is supplied — standard white plastic chain or optional steel chain. The advisor selects the chain type; the Accessory Engine and Supplier Capability Engine resolve the compatible product/SKU and pricing.

Supplier documents determine availability.

## 17. Child Safety

The shared Child Safety Engine implements BS EN 13120 compliance (MCD-01A §5.6). "Installation height" and "Customer refusal workflow" are shared responsibilities of that engine and are not restated here.

Vertical-specific implementation is limited to:
- **Chain/cord safety device mapping** — which device applies depends on Control Type: Cord & Chain requires operating-chain safety devices (per MCD-01A §5.6); Wand's tilt mechanism has no operating cord loop. **Not yet specified:** whether Draw Configuration (Wand Left/Right/Split Draw, or Left/Right Stack/Split Draw under Cord & Chain) introduces its own separate cord requiring safety devices regardless of Control Type — needs supplier/business data before technical design (see §23).
- **Chain length calculation basis** — where an operating or stabilising chain is present, its length is drop-derived, consistent with MCD-01A §3's explicit note that chain length is a Child Safety Engine calculation, not a Configuration input. **Not yet specified:** the actual formula or supplier-defined ratio.
- **Breakaway device compatibility** with the selected chain/cord safety device and Construction Variant.

## 19. Recommendation Rules

Recommendations are provided by the shared Recommendation Engine. Vertical-specific recommendations may include:
- Wand control for households prioritising cordless/child-safety-first operation
- Motorised tilt for high or hard-to-reach windows
- Split Draw for wide window openings
- Cassette-equivalent premium headrail (Vogue) where aesthetics require
- Alternative suppliers where lead time or Curved/Sloping capability is improved

Recommendations shall explain the reason for the recommendation. Recommendations shall never override advisor decisions.

## 20. Supplier Capability Requirements

Supplier documents shall define: available constructions (including Curved/Sloping), headrail colours, component colour-matching (wand/cord/bracket/end cap), bracket compatibility, motor compatibility, chain/bottom-weight pricing and colour availability, size limits, manufacturing limits, premium options, technical dimensions, lead times, and the 127mm Legacy/Future louvre width capability record (see §4).

**Confirmed data point:** Motorised (Tilt Only) on Vogue is subject to a maximum width of 3m and a minimum width of 200mm where an open cassette is used — Supplier Capability data specific to that Control Type and Construction Variant combination, not a general Vogue width limit. See §8 and §23 Decision Register for source evidence.

Supplier capability shall remain independent of Product Engine behaviour.

## 21. Product Configuration Summary

This section summarises the complete advisor workflow as the implementation blueprint for the UI. Each step is labelled **[Advisor]** (Advisor Selection), **[Derived]** (Derived Default — no question shown), or **[Supplier]** (Supplier Capability data consumed, not an advisor-facing step).

Sales MVP applies 89mm louvre width as a system default. It is not an advisor-facing workflow step. The deliberate exclusion of 127mm is documented in §4, §8, and the Decision Register.

1. **[Advisor]** Construction Variant (Slimline / Nova / Cruze / Vogue / Curved / Sloping)
2. **[Advisor]** Recess / Exact Fit, Window Width, Window Drop, Installation Height
3. **[Advisor]** Control Type (Wand / Cord & Chain / Motorised (Tilt Only))
   - If Wand → **[Advisor]** Draw Configuration only (Wand Left / Wand Right / Split Draw) — control position and stack direction are physically coupled (Wand Left = positioned left and stacks left; Wand Right = positioned right and stacks right); no separate Control Side question. **[Supplier]** single vs. two-wand implementation for Split Draw
   - If Cord & Chain → **[Advisor]** Control Side (Left / Right); **[Advisor]** Draw Configuration (Left Stack / Right Stack / Split Draw); **[Advisor]** Chain Type (White Plastic Standard / Steel Chain Upgrade)
   - If Motorised (Tilt Only) → **[Advisor]** Motor Left / Motor Right. **[Supplier]** Draw mechanism: for Vogue, Wand automatically (confirmed Supplier Capability rule; no advisor question); for other Construction Variants, per Supplier Capability (§20). See §23 Decision Register for full evidence.
4. **[Advisor]** Headrail Colour
   - **[Derived]** Wand colour, Cord colour, Bracket colour, end caps
   - **[Supplier]** available colour range per Construction Variant
5. **[Advisor]** Bracket Type (Face Fix / Top Fix / Extension)
   - **[Derived]** Bracket colour (from step 4)
   - **[Supplier]** physical bracket eligible for the combination of Bracket Type + Construction Variant
6. **[Advisor]** Fabric Selection, Colourway
7. **[Advisor]** Bottom Weight Colour (White Standard / Black Upgrade)
8. **[Advisor]** Bottom Chain (White / Black / No Chain — single selection)
   - If No Chain → **[Derived]** chainless bottom weight system substituted automatically
9. **[Advisor]** Accessories (remote controls, wall switches, smart hubs — Motorised tilt only, where applicable)
